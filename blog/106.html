sts를 이용한 github 저장소 업로드

이번엔 sts에서 바로 Githup으로 프로젝트 업로드 방법을 알아봅시다.

예제 프로젝트

예제 프로젝트는 이곳에서 확인하세요.

Github Project

Git Configuration

일단 해당 프로젝트에서 [Team] - [Share Project]를 선택하고 Git 타입으로 선택해서 로컬 저장소도 대충 만들어줍니다. 그러면 Package Explorer 윈도우에 [git NO-HEAD]라는 단어가 큼직하게 박힙니다.

Identify Yourself

만약 기본적인 Git 설정을 하지 않았다면 Full Name 과 E-mail을 입력하는 윈도우가 나옵니다. 적절하게 입력해줍니다. 그러면 Commit Changes 윈도우가 나옵니다.

Commit Changes

Commit Message 및 하단의 Files에서 커밋할 파일들을 선택해줍니다. 그 후 Commit and Push 버튼을 클릭합시다.

Push Branch master

커밋하기 위한 서버의 정보를 요구합니다. 다음처럼 입력해줬습니다.

Remote Name : origin
Location
URI : https://github.com/YOUR_ID/springbootdemo.git
Host : github.com
Repository path : /YOUR_ID/springbootdemo.git
Connection : https
Port : (없음)
Authentication
User : XXXXX
Password : XXXXX

* github 저장소는 미리 만들어 둡시다.

이후 [Next]를 계속 클릭하고 [Finish] 버튼까지 클릭합니다.

프로젝트 rejected 오류가 나는 경우에는 [Team] - [Pull] 을 한번 실행해줍니다.

프로젝트 확인

https://github.com/YOUR_ID/springbootdemo에 demo 프로젝트를 업로드 했습니다. :)
