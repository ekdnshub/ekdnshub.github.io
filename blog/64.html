JavaScript Objects

자바스크립트에서 객체Object들은 킹king이다. 만약 이 글을 보고 있는 당신이 객체들을 이해한다면 당신은 자바스크립트를 안다고 할 수 있다.

자바스크립트에서 대체적으로 "어느것이든" 객체다.

  • Boolean, Number, String 타입은 객체가 될 수 있다. 또는 객체로써 기본 자료형 취급한다.
  • Date, Math, Regular expression정규표현식, Array배열, function함수 타입은 언제나 객체다.
  • Object 타입은 객체다.

자바스크립트에서 기본 자료형을 제외한 모든 값들은 객체들이다.

기본 자료형이란 다음과 같다.

  • 문자열strings : "JDM"
  • 숫자numbers : 3.14
  • true, false, null, undefined

Objects are Variables Containing Variables

객체들은 변수들을 포함하는 변수들이다.

자바스크립트 변수들은 단일 값을 가질 수 있다.

var person = "JDM";

객체들은 변수와도 같다. 그러나 객체들은 많은 변수들을 가질 수 있다.

이 값들은 name:value 형태의 한 쌍으로 쓰여질 수 있다. name 과 value는 콜론 기호로 분리 된다.

var person = {firstName:"DM", lastName:"J", age:29, eyeColor:"black"};

자바스크립트 객체는 변수(named values라 불리는)들이 정렬 되지 않은 집합이다.

Object Properties

자바스크립트 객체에서 named values는 속성properties이라고도 한다.

Property Value
firstName DM
lastName J
age 29
eyeColor black

객체들은 name value 한 쌍으로 유사하게 다음과 같은 곳에서 유사하게 쓰인다.

  • PHP에서 Associative arrays
  • 파이썬Python에서 사전 자료형Dictionaries
  • C언어에서 해시 테이블Hash tables
  • 자바Java에서 해시 맵Hash maps
  • 루비Ruby 또는 펄perl에서 해시hashes

Object Methods

메소드Method란 객체에서 수행될 수 있는 행동들actions이다.

객체 속성은 기본 자료형primitive values, 다른 객체, 그리고 함수들functions이 가능하다.

객체 메소드Object Methods는 함수를 포함한 객체 속성이다. 아래는 하나의 객체를 표현한 것이다.

Property Value
firstName DM
lastName J
age 29
eyeColor black
fullName function() {return this.firstName + " " + this.lastName;}

TIP.
자바스크립트 객체는 named values, 속성 및 메소드들을 위한 컨테이너다.

Creating a JavaScript Object

자바스크립트에서 사용자는 사용자 소유의 객체들을 만들거나 정의할 수 있다. 새로운 객체를 만드는 방법들은 다양하다.

  • 단일 객체Single object를 정의하거나 만들때는 객체 리터럴Object literal을 사용한다.
  • 또는 단일 객체를 만들기 위해 new 키워드를 사용한다.
  • 객체 생성자를 정의해서 구성할 수 있다.

ECMAScript 5에서는 객체를 만들때 function() Object.create()를 사용할 수 있다.

Using an Object Literal

자바스크립트 객체를 만들기 위한 쉬운 방법이 있다. 객체 리터럴을 사용하면 사용자는 하나의 구문에서 객체를 만들거나 정의한다.

객체 리터럴은 예를 들자면 {age:50} 같은 중괄호 안에 name:value 값을 넣어 구성된 목록이다.

다음의 예제를 보자, 4개의 속성을 가진 새로운 자바스크립트 객체를 만드는 것이다.

var person = {firstName:"DM", lastName:"J", age:29, eyeColor:"black"};

공백Space과 줄의 분리line breaks는 중요하지 않다. 객체 정의는 여러 줄에 걸칠 수도 있다.

var person = {
	firstName:"DM",
	lastName:"J",
	age:29,
	eyeColor:"black"
};

Using the JavaScript Keyword "new"

다음의 예제 또한 4개의 속성을 가진 새로운 자바스크립트 객체를 만든다.

var person = new Object();
person.firstName = "DM";
person.lastName = "J";
person.age = 29;
person.eyeColor = "black";

new 키워드를 사용하는것과 객체 리터럴을 사용하는 예제들은 완벽하게 같은 동작을 한다. new Object()를 사용하는 것은 필요하지 않다. 간단하게, 읽기 좋게, 실행 속도를 빠르게 하기 위해서는 객체 리터럴 메소드를 사용하라.

Using an Object Constructor

지금까지의 예제는 많은 상황을 제한하고 단지 단일 객체만을 만든다. 때때로 우리는 많은 객체들의 타입으로 이용될 "객체 타입"을 가지는 것을 선호한다.

객체 타입을 만들기 위해 흔한 방법으로 객체 생성자 함수object constructor function를 사용하는 것이다.

function person(first, last, age, eyecolor) {
    this.firstName = first;
    this.lastName = last;
    this.age = age;
    this.eyeColor = eyecolor;
}
var myFather = new person("DM", "J", 29, "black");
var myMother = new person("HR", "L", 3, "green");

여기서 person() 함수는 객체 생성자다.

사용자가 객체 생성자를 하나 만들면, 사용자는 같은 타입의 새로운 객체들을 만들어 낼 수 있다.

The "this" Keyword

자바스크립트에서 this라 불리는 것은 자기 자신을 뜻하는 자바스크립트 코드이자 객체다.

함수 내에서 this의 값은 함수 "자신"의 객체다.

객체 내에서 this의 값은 객체 "자신"이다.

객체 생성자object constructor에서 this 키워드는 값을 가지지 않는다. 이것은 단지 새로운 객체를 대신하는 것이다.

생성자가 객체를 만들때, this의 값은 새로운 객체가 될 것이다.

Warning

this는 변수가 아니다. 이것은 "키워드"다. 사용자는 this의 값을 바꿀 수 없다.

Built-in JavaScript Constructors

자바스크립트는 객체를 만들 수 있는 기본 생성자를 가지고 있다.

var x1 = new Object();    // 새로운 Object 객체
var x2 = new String();    // 새로운 String 객체
var x3 = new Number();    // 새로운 Number 객체
var x4 = new Boolean()    // 새로운 Boolean 객체
var x5 = new Array();     // 새로운 Array 객체
var x6 = new RegExp();    // 새로운 RegExp 객체
var x7 = new Function();  // 새로운 Function 객체
var x8 = new Date();      // 새로운 Date 객체

Math() 객체는 이 목록에 포함되지 않는다. Math는 전역 객체global object기 때문이다. new 키워드는 Math 안에서 사용할 수 없다.

이건 알고 있나요?

보다시피, 자바스크립트는 기본 데이터 형인 String, Number, Boolean의 객체 버전이다.

복잡한 객체는 만들 이유가 전혀 없다. 기본 데이터 형 값들은 빠른 속도로 실행된다.

그리고 new Array() 또한 사용할 이유가 없다. 대신 배열 객체를 위해서 [] 를 사용하라.

그리고 new RegExp() 또한 사용할 이유가 없다. 대신 리터럴 패턴인 /()/ 를 사용하라.

그리고 new Function() 또한 사용할 이유가 없다. 대신 표현형인 function(){} 을 사용하라.

그리고 new Object() 또한 사용할 이유가 없다. 대신 객체 리터럴인 {} 을 사용하라.

var x1 = {};            // 새 객체 타입
var x2 = "";            // 새 기본 문자열 타입
var x3 = 0;             // 새 기본 숫자 타입
var x4 = false;         // 새 기본 boolean 타입
var x5 = [];            // 새 기본 배열 타입
var x6 = /()/           // 새 기본 정규식 타입
var x7 = function(){};  // 새 기본 함수 타입

JavaScript Objects are Mutable

객체는 변경할 수 있다. 객체는 by reference로 참조 가능하다. by value로 참조 하지 않는다.

만약 y가 객체라면 밑에 기술된 구문은 y를 복사해서 가지지 않는다.

var x = y;  // 이것은 y를 복사해서 새로운 객체를 만들지 않는다.

객체 x는 y를 복사한 값을 가지지 않는다. 이것은 y일 뿐이다. x와 y가 가리키고 있는 것은 같은 객체다.

어떠한 변경이라도 y를 변경하면 x도 같이 변경된다. x와 y는 같은 객체를 바라보고 있기 때문이다.

var person = {firstName:"DM", lastName:"J", age:29, eyeColor:"black"}
var x = person;
x.age = 10; // 이것은 x.age를 변경했지만 x.age와 person.age의 값을 전부 변경한다.
